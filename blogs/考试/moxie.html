    # 第一章

    # hadoop 优点

    低成本：低成本计算机集群环境，分布式处理大数据

    高可靠：自动维护数据文件，防止丢失

    高容错：某个宕机，自动转发给其他计算机处理

    高效率：可并列计算、移动计算

    高扩展性：可随时增加更多计算机集群

    # 第二章

    # 简述独立模式、为分布式模式、完全分布式模式部署 hadoop 区别

    一台计算机 jvm 进程中模拟 hadoop 集群工作模式

    一台计算机的不同 jvm 进程运行 hadoop

    多台计算机的 jvm 运行 hadoop

    # 第三章

    # 1、简述 HDFS 的健壮性

    （1）心跳机制

    运行期间 namenode 和每一个 datanode 建立心跳机制，每隔一个周期（默认 3 秒）给 namenode 发送心跳信息

    （2）副本机制

    将 datanode 的 block 存副本，默认三个（包含本身）

    （3）数据完整性校验

    读取文件时，对文件每一个 block 校验和，保存为隐藏文件，获取文件后会校验匹配

    （4）安全模式

    只能对 HDFS 读，不能写。

    （5）快照

    HDFS 整个文件系统或某个目录在某个时间点的镜像

    # 2、简述 HDFS 写文件的流程，设文件大小为 300MB。

    1、客户端上传 1.txt 文件到指定目录的请求，与 NameNode 建立通信

    2、namenode 检查是否有权限、已经文件是否存在，通过->通知客户端上传，并且发分块策略

    3、客户端根据分块策略进行切分

    4、向 namenode 请求上传第一个 block

    5、namenode 根据副本机制和机架感知返回可上传 blk1 的 DataNode 列表

    6、namenode 接收到 blk1 上传的 datanode 列表后,就近选择 hadoop1,hadoop1 会与 hadoop2 建立管道，hadoop2 会与 hadoop3 建立管道

    7、从 hadoop3-hadoop1 汇报建立成功

    8、开始传输：

    讲 blk1 写入内存

    讲 blk1 划分 packet(64kb)

    发送第一个分块给 Hadoop1

    # 第四章

    # 简述 Shuffle 工作原理

    # MapTask 中的 Shuffle：

    1、MapTask 处理结果放内存缓存区、(默认 100MB)超过 80%就创建并写入临时文件

    2、写入要确认 reducetask 数量、一个对应一个分区、作用：不倾斜

    3、数据排序、combiner 就合并排序后的结果、减少 Copy 阶段复制的数据和数据量

    4、处理临时文件放入已分区文件 作用：Copy 阶段少复制

    5、输出到对应 reducetask

    # ReduceTask 中的 Shuffle：

    1.copy 阶段，从不同处理结果复制数据，写入内存缓冲区

    2、到达阀值，合并排序，写入临时文件

    3、临时文件越来越多，在合并排序生成新的文件

    # 简述 YARN 基本架构的组成部分及其作用。

    在 YARN 基本架构中，分别由 ResourceManager、ApplicationMaster、NodeManager 和 Container 组成，

    ResourceManager 为全局的资源管理器，负责整个系统的资源管理和分配；

    ApplicationMaster 为每个应用程序特有的，负责单个应用程序的管理；

    NodeManager 负责在结点上启动和管理 Container（容器）；

    Container 封装了每个应用程序所使用的资源。

    # 第七章

    # 简述数据仓库分层中源数据层、数据仓库层和数据应用层存储的数据。

    （1）源数据层 源数据层存储的数据是数据仓库的基础数据，该层存储的数据抽取自不同的数据源，抽取的这些数据通常会进行诸如去噪、去重、标准化等一些列转换操作后才会加载到源数据层。

    （2）数据仓库层 数据仓库层存储的数据是对源数据层中数据的轻度汇总，所谓轻度汇总就是按照一定的主题去组合这些数据。

    （3）数据应用层 数据应用层的数据主要是提供给数据分析、数据挖掘、数据可视化等实际业务场景使用的数据。

    # 2、简述 Hive 中分区和桶的作用。

    分区：

    根据指定分区规则，整体数据划分多个单独数据存储，每个单独数据看作一个分区，存储不同目录

    桶：

    根据指定分桶规则，将表随机、均匀划分到不同的桶存储，每个桶存在 hdfs 不同文件中，有效避免数据倾斜（分区的缺点）
